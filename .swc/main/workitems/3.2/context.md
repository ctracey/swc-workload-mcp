## Pass 2 — 2026-05-26 (review feedback: swap to real-stdio transport)

- **Feedback driver:** the Phase A in-memory client bypasses real stdio framing. The suite as built proves the layers from `tools.py` down to `workload.json` are correct, but says nothing about the stdio transport that real MCP clients use. Decision (per `feedback.md`): swap the conftest harness to spawn a real server subprocess via `mcp.client.stdio.stdio_client(StdioServerParameters(...))` and drive it over real stdio.
- **Decision:** **session-scoped server subprocess**, one for the whole pytest run. Tests share a single `ClientSession`. Per-test workload isolation stays via per-test `tmp_path` — workload state never crosses tests because every tool call carries the per-test folder path; the server itself holds no per-test state.
- **Decision:** `anyio_backend` promoted from function to session scope so the session-scoped async fixture can live across tests. Required by anyio's pytest plugin — the session-scoped event loop is shared with the per-test async fixtures (`mcpw_ready`) and async tests.
- **Decision:** `mcpw` becomes a sync fixture that simply composes the per-test paths with the shared session into a callable closure. Only `mcpw_ready` stays async (it awaits the `init` call to set up the fresh workload).
- **Decision:** `SWC_WORKLOAD_BIN` moves from per-test `monkeypatch.setenv` to `StdioServerParameters.env={**os.environ, "SWC_WORKLOAD_BIN": _REAL_CLI_PATH}`. Set once when the server subprocess spawns; inherited by every bridge → CLI subprocess inside it. Same determinism guarantee, fewer moving parts.
- **Tried:** considered per-test server subprocess (one spawn per test) for strict isolation. Rejected — adds ~50-200ms × 70 tests of pure overhead for no isolation gain. Workload state is already per-test; the server has no per-test state to leak.
- **Tried:** considered the SDK's `pytest-asyncio` compatibility shim for session-scoped async fixtures. Not needed — anyio's `@pytest.fixture(scope="session")` on an async generator works directly when `anyio_backend` is also session-scoped.
- **Test results:** 70 passed, 7 skipped, 0 failed against the new stdio transport — identical breadth to Pass 1. Runtime: 13.00s for the integration files alone (vs 12.75s on the in-memory harness — ~0.25s overhead for the subprocess + handshake spread across 70 tests). Full suite: 127 passed, 7 skipped in 14.52s.
- **Demo-bug-mirror test (`test_add_as_child_of_parent`) still passes** under real stdio. The bug is now confirmed not in our tool layer AND not in stdio framing — almost certainly in the MCP client's form-input → JSON serialisation (Inspector's domain) or in the user-input handling at the host layer.
